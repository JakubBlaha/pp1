I was saying that a write must cause any subsequent read operations to have the same value as was written.

When the system reads the registers A and B and their sum is greater than 200, the system needs to enter the emergency mode state within 200ns.

That means a test case covering this would need to:
 - write to register A, write to register B

The checker would then know if we have covered the case where their sum is greater than 200.
This means that the checker somehow knows that that the write would cause subsequent reads to to have the values, so it would know that we are testing or not testing the threshold 200.

Requirement:
G(fired(E) => (x > 200) => set_state(EMERGENCY))
