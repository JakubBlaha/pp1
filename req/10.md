```
The software shall enter the emergency mode if any of the following failures individually
occurs twice in less than 10 seconds:
a) sensor failure
b) energy failure
c) communication failure.
```
```
ENTITY EmergencyMode : STATE  // emergency operating mode

REQ 10 [State-machine with timed event counting]
  TIER:    MTL
  TRIGGER: ANY_OF {
    ALL_OF { COUNT(ON sensor_failure) >= 2, INTERVAL(ON sensor_failure, 2) < 10s },
    ALL_OF { COUNT(ON energy_failure) >= 2, INTERVAL(ON energy_failure, 2) < 10s },
    ALL_OF { COUNT(ON comm_failure)   >= 2, INTERVAL(ON comm_failure,   2) < 10s }
  }
  EFFECT:
    SET_STATE EmergencyMode

// MTL
G( ((COUNT(ON sensor_failure) >= 2 ∧ INTERVAL(ON sensor_failure, 2) < 10s)
  ∨ (COUNT(ON energy_failure) >= 2 ∧ INTERVAL(ON energy_failure, 2) < 10s)
  ∨ (COUNT(ON comm_failure)   >= 2 ∧ INTERVAL(ON comm_failure,   2) < 10s))
   → F( in_state(EmergencyMode) ) )
```

# Test Cases

## English

```
TC1) No failure occurs -> expected: not entering emergency mode 
TC2) Sensor failure twice in interval of 9 seconds -> expected: entering emergency mode 
TC3) Sensor failure twice in interval of 11 seconds -> expected: not entering emergency mode 
TC4) Energy failure twice in interval of 9 seconds -> expected: entering emergency mode 
TC5) Energy failure twice in interval of 11 seconds -> expected: not entering emergency mode 
TC6) Comm failure twice in interval of 11 seconds -> expected: not entering emergency mode
TC7) Energy failure occurs and then Comm failure in an interval of 9 seconds -> expected: not entering emergency mode (robustness test case)
```
