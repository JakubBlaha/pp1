```
The software must configure IVORx registers with the addresses of Exception Vectors that
compose the Exception Vector Table, according to the following table:

IVORx Exception
0 Critical input
1 Machine check
```

```
ENTITY IVOR0         : STORAGE  ATTRIBUTES { REGISTER }
ENTITY IVOR1         : STORAGE  ATTRIBUTES { REGISTER }
ENTITY CriticalInput : STORAGE  ATTRIBUTES { MEMORY }
ENTITY MachineCheck  : STORAGE  ATTRIBUTES { MEMORY }

REQ 04 [Interrupt / exception handling – vector table config]
  TIER:     LTL
  MODALITY: EVENTUAL
  EFFECT:
    STORE ADDR(CriticalInput) TO IVOR0
    STORE ADDR(MachineCheck)  TO IVOR1

// LTL
F( value_at(IVOR0) = ADDR(CriticalInput) ∧ value_at(IVOR1) = ADDR(MachineCheck) )
```
